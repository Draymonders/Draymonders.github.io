正文

Java语言概述

Java语言是跨平台的，JVM不是跨平台的
JavaSE –桌面开发
JavaME –安卓开发
JavaEE –Web开发
JVM：保证Java语言跨平台
JRE：Java程序的运行环境 (JVM+类库
JDK：Java程序的开发环境 (JRE+工具

Java程序的开发执行流程：
A:编写java源程序(.java)
B:通过javac命令编译生成.class文件
C:通过java命令运行.class文件
path环境变量里面记录的是可执行性文件，如.exe文件，对可执行文件先在当前路径去找
如果没找到就去path环境变量中配置的路径去找
classpath环境变量里记录的是java类的运行文件所在的目录

基础语法

关键字

全部小写 (String 不是关键字

标识符

A:英文大小写字母
B:数字
C:$和_

常见命名规则

A:包 全部小写
单级包：小写
举例：package,com
多级包：小写，并用.隔开
举例：com.baidu
B:类或者接口
一个单词：首字母大写
举例：Student,Demo
多个单词：每个单词首字母大写
举例：HelloWorld,StudentName
C:方法或者变量
一个单词：首字母小写
举例：name,main
多个单词：从第二个单词开始，每个单词首字母大写
举例：studentAge,showAllNames()
D:常量
全部大写
一个单词：大写
举例：PI
多个单词：大写，并用_隔开
举例：STUDENT_MAX_AGE

常量

A:字面值常量

B:自定义常量
以后补充
进制转换

数据类型

变量

注意变量的默认值，以及作用域，以及赋初值
A:基本数据类型：4类8种
整型 byte,short,int,long
浮点数 float double
字符 char
布尔 boolean
B:引用数据类型：类，接口，数组。

注意类型

A:下面两种方式有区别吗?
float f1 = 12.345f; //这个是本身就是float
float f2 = (float)12.345; //这个是double转float，丢精度
B:下面的程序有问题吗，如果有，在哪里呢?
byte b1 = 3;
byte b2 = 4;
byte b3 = b1 + b2; //变量运算会提升，所以有问题
byte b4 = 3 + 4; //这个是常量，先把结果算出来，看有没有在byte类型中
C:下面的操作结果是什么呢?
byte b = (byte)130; //数据运算都是补码运算的
D:字符串参与运算
这里其实是字符串的连接

System.out.println("hello"+'a'+1); //helloa1
System.out.println('a'+1+"hello"); //98hello
System.out.println("5+5="+5+5); //5+5=55
System.out.println(5+5+"=5+5"); //10=5+5

运算符

算术运算符，赋值运算符，比较运算符，逻辑运算符，位运算符，三元运算符

  1. instanceof(检查是否是类的对象

  2. trick(一个数据对另外一个数据异或两次 结果不变，相同的数据异或为0 然后一个数据异或0还是本身)

  3. 异或实现两个数交换
    a = a ^ b;
    b = a ^ b; // a ^ b ^ b,然后b = a
    a = a ^ b; // a ^ b ^ a,然后a = b

  4. 所有的三元运算符能够实现的，if语句的第二种格式都能实现。 反之不成立。

控制语句

JDK7以后可以 枚举,字符串,byte,short,int,char类型或变量，不能long
if,switch,for,while,do while,continue,break 大概没什么好写的

方法

A:方法不调用不执行
B:方法之间是平级关系，不能嵌套定义
C:方法定义的时候，参数是用，隔开的
D:方法在调用的时候，不用在传递数据类型
E:支持方法的重载 即在同一个类中，方法名相同，参数列表不同。

数组

堆和栈的区别

栈：
数据使用完毕，就消失。
堆：
A.每一个new出来的东西都有地址
B.每一个变量都有默认值
byte,short,int,long 0
float,double 0.0
char ‘\u0000’
boolean false
引用类型 null
C.数据使用完毕后，在垃圾回收器空闲的时候回收。

格式

参数传递

Java中只有值传递。
基本类型：形式参数的改变不影响实际参数
引用类型：形式参数的改变直接影响实际参数
