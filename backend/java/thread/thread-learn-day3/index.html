多线程学习 day3

发表于 2019-02-16   |   594 字   |   2 min
等待队列，volatile, 缓存一致性，happens-before原则，不可变对象，ThreadLocal等多线程基础知识

wait set(也可以理解为等待队列)

1.所有的对象都会有一个wait set,用来存放调用了该对象wait方法之后进入block状态线程
2.线程被notify之后，不一定立即得到执行
3.线程从wait set中被唤醒顺序不一定是FIFO.
4.线程被唤醒后，必须重新获取锁

volatile

验证volatile内存可见性和有序性.

解决缓存一致性问题

  1. 给总线加锁(数据总线，指令总线，控制总线)
  2. 缓存一致性协议.(进行update操作时候，发现其他cpu/cache有副本，则会发通知给其他cpu/cache失效)

并发编程 三个重要的概念

  1. 原子性(atomic)
  2. 可见性(visible)
  3. 有序性(order)

happens-before relationship

  1. 程序顺序原则： 一个线程内保证语义的串行性
  2. volatile：volatile的写，先发生于读
  3. 锁规则：unlock 必然发生在随后的lock前
  4. A先于B，B先于C，那么A必然先于C
  5. 线程的start()方法先于它的每一个动作
  6. 线程的所有操作优先于线程的终结(Thread.join())
  7. 线程的中断(interrupt) 先于被中断线程的代码
  8. 对象构造函数执行，先于finalize()方法

volatile关键字

一旦一个共享变量被volatile修饰，具备两层语义

  1. 保证了不同线程间的可见性
  2. 禁止对其进行重排序，保证有序性
  3. 并未保证原子性

使用场景

  1. 状态标记
  2. 屏障前后的一致性

不可变对象

  1. 不可变对象一定是线程安全的
  2. 可变对象不一定是线程不安全的

如何保证不可变对象

  1. 变量都是private final修饰的
  2. 类不可被继承 即class被final修饰
  3. 不提供set接口.
  4. 成员变量是引用类型的,要使用get,要么用Collections.unmodify转为不可变，要么clone方法，拷贝一个
