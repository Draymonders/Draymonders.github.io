# 代码之丑

粗读了极客时间的《代码之丑》，大概有如下原则

1. 命名
2. 重复代码（Don't repeat yourself.）
3. 长函数
4. 大类
5. 长参数列表（动态参数用传入的方式，不变参数放到类的内部）
6. 嵌套if，提前返回
7. 相同的switch条件判断，进行接口抽象
8. 缺乏封装，导致重复代码乱飞
9. 限制可变的数据
10. 一次性完成变量的初始化

--- 没理解深刻的原则 ---

11. 依赖混乱，接口要稳定，实现可以变
12. 保持代码在各个层面上的一致性
13. 不断学习“新”的代码风格，不断改善自己的代码
14. 代码评审暴露的问题越多越好，频率越高越好
